# Game Enrichment Pipeline

## What this project does
Extracts Themes and Main Features from MGA game PowerPoint decks, standardises them
against a market-aligned SEO taxonomy, resolves localisation variants, and outputs
a single enriched CSV ready for casino site SEO use.

## Project structure
```
game-enrichment/
├── CLAUDE.md                        ← you are here — read this first, always
├── main.py                          ← CLI entry point, orchestrates all phases
├── config/
│   ├── seo_taxonomy.json            ← canonical theme + feature tags (English, SEO-ready)
│   ├── market_names.xlsx            ← master localisation DB (700 rows, 357 game families)
│   └── processed_market_names.json  ← pre-processed lookup cache (generated by main.py)
├── agents/

│   ├── feature_classifier.py        ← Phase 2b: Claude API — maps to SEO feature tags
│   ├── localisation_resolver.py     ← Phase 2c: deterministic lookup, Claude for fallback
│   └── consolidator.py             ← Phase 3: merges subagent outputs, flags low confidence
├── data/

└── output/
    ├── games_enriched.csv           ← final output: one row per game family
    └── review_flagged.csv           ← subset: rows needing human review
```

## Input data

### PPTX folder structure
```
\CATEGORY\GAMENAME\[various .pptx files]
```
- Category maps to the `category` column in market_names.xlsx (SLOTS5, SLOTS3, MEGAWAYS, BINGO, etc.)
- GameName folder segment is used for lookup against `name` and `COMMERCIAL NAME` columns
- For each game folder: prefer the file whose name contains "Comercial" (case-insensitive)
- If no "Comercial" file exists: take the first .pptx found
- Path format: Windows (\\server\share\... or C:\...) — use pathlib.Path throughout for cross-platform safety

### market_names.xlsx columns
| Column | Description |
|--------|-------------|
| name | Internal name (e.g. "AbigailNl") — has market suffix |
| category | SLOTS5, SLOTS3, MEGAWAYS, BINGO, ROULETTE, EXTERNAL |
| COMMERCIAL NAME | Localised display name for that market |
| MARKET | SPAIN, PORTUGAL, .COM, NETHERLANDS, ITALY, COLOMBIA, CANADA, SWEDEN |
| Gameid | Numeric game ID |
| tablename | Internal DB key (e.g. "Slots5GeneralCountersAbigailNl") |
| Enum | Internal enum (e.g. "SLOTS5_ABIGAIL_NL") |
| Deactivated | YES or NO |
| rate | Pro or Premium |

### Game family grouping
Strip market suffix from `tablename` to get the base key:
```python
import re
suffix_pattern = re.compile(r'(NoIp|Es|Pt|It|Co|Nl|Ca|Se)$', re.IGNORECASE)
base_key = suffix_pattern.sub('', tablename)
```
Games sharing a base key are variants of the same product.
- Spain entry (no suffix, or Es suffix) is the canonical / source-of-truth entry
- Deactivated=YES rows: skip unless --include-deactivated flag is passed
- 225 games are Spain-only (no other market variant) — still valid, just no localisation to resolve

### IP / celebrity detection
Canada (Ca) and Sweden (Se) variants often attach celebrity IPs to base games:
- "Charlie Riina Alchemy Riches" (Ca) vs "Alchemy Riches" (base)
- Detect by comparing Ca/Se commercial name against base commercial name
- If different: extract the prefix as the celebrity name, add as a theme tag alongside "Celebrities"

### review_flagged.csv
Same schema as games_enriched.csv, filtered to review_flag=True rows only.
Human edits this file, then re-runs consolidator with --merge-review flag.

## Confidence thresholds
| Signal | Flag threshold | Notes |
|--------|---------------|-------|
| Theme classifier | < 0.75 | Claude API confidence score |
| Feature classifier | < 0.75 | Claude API confidence score |
| Localisation fuzzy match | < 0.75 | rapidfuzz ratio |
| No PPTX found | always flag | pptx_found=False |
| No market DB match | always flag | localisation_match_confidence=0 |

## Classifier agent design

### Theme classifier system prompt (use verbatim)
```
You are an SEO taxonomy classifier for casino slot games.
Given the extracted text from a game's PowerPoint presentation, identify which theme tags apply.

Available theme tags (assign ALL that apply — games typically have 2–6 themes):
{themes_list}

Rules:
- Tags must come from the approved list only. Do not invent new tags.
- Input text is in Spanish. Use the spanish_aliases lookup to map Spanish terms before classifying.
- If a celebrity/real person is mentioned: add "Celebrities" tag + the person's full name as a separate tag + any relevant sport/genre tag.
- If a film/TV genre is referenced (heist, western, horror etc.): add "Movies & TV" tag AND the genre tag.
- Respond ONLY with valid JSON: {"themes": ["Tag1", "Tag2"], "confidence": 0.85, "reasoning": "one sentence"}
- Confidence should reflect how clearly the text supports the classification (0.5 = guessing, 0.9+ = explicit mention).
```

### Feature classifier system prompt (use verbatim)
```
You are a casino game mechanics classifier.
Given the extracted text from a game's PowerPoint presentation, identify which feature/mechanic tags apply.

Available feature tags (assign ALL that apply):
{features_list}

Spanish-to-English alias map:
{spanish_aliases_features}

Rules:
- Tags must come from the approved list only. If you encounter a mechanic not on the list, add it to the "unknown_features" array for human review — do not drop it.
- Input text is in Spanish.
- "Compra Free Spins" or "Opción de compra" always maps to "Buy Feature" / "Buy Free Spins".
- Megaways always maps to "Megaways".
- Respond ONLY with valid JSON: {"features": ["Tag1", "Tag2"], "unknown_features": ["any not on list"], "confidence": 0.85, "reasoning": "one sentence"}
```

## Phase execution order
1. `python main.py extract --input "C:\path\to\pptx\folder"` — Phase 1 only, no API calls
2. `python main.py classify --dry-run` — Phase 2, processes first 5 games only (for testing)
3. `python main.py classify` — Phase 2, full run (parallel subagents)
4. `python main.py consolidate` — Phase 3, merge + flag
5. `python main.py merge-review` — after human edits review CSV, re-merge

## Key implementation notes
- Use `pathlib.Path` everywhere — not `os.path` — for Windows path compatibility
- Use `asyncio` + `asyncio.gather` for parallel subagent API calls in Phase 2
- Rate limit: max 10 concurrent Claude API calls (ANTHROPIC_API_KEY from env)
- Model: claude-sonnet-4-5 for classifiers (cost/quality balance for bulk classification)
- Never hardcode the PPTX input path — always take from CLI argument or prompt
- The `data/raw_extracts/` and `output/` dirs are gitignored — do not commit game data
- Log every API call result to `data/classifier_log.jsonl` for debugging

## Common Spanish terms in MGA PPTXs
- "CATEGORIAS PRODUCTO" / "VENTA – CATEGORIAS PRODUCTO" → slide 2, primary source
- "ARGUMENTOS DE VENTA" → slide 3, secondary source for feature confirmation
- "Compra Free Spins" / "Opción de compra" → always = Buy Feature
- "Sorteo de Free Spins" → Free Spins (with random spin count mechanic)
- "Localización (Audios y textos)" → localisation note, not a feature tag
- "Internacional MGA" → distribution scope note, not a feature tag
- "DESARROLLO" section → internal dev notes, skip entirely
- "*Gráficos no finales" / "*Gráficos cogidos de internet" → disclaimer, skip
- Numbers alone (1, 2, 3, 4) → slide layout numbering, skip

## Session workflow commands

### catchup (run at start of every session)
1. Read `dev/projectlog.md` — note current status and outstanding items
2. Read `dev/ref/architecture.md` — refresh on pipeline design
3. Read any stage summary files relevant to the current session
   (stage1-summary.md before Session 2, stage2-summary.md before Session 3)
4. Confirm current status to user in 3–5 sentences
5. Suggest the immediate next step

### record-session (run at end of every session)
1. Append a new session entry to `dev/projectlog.md`:
   - Keep entries brief — what was done, what's outstanding, any blockers
   - Detailed technical notes go in `dev/ref/` not the log itself
   - Update the token usage table
2. Write or update relevant `dev/ref/` file for technical detail
3. Check with user for plans for next session — add to log
4. Git commit and push to main
5. Confirm to user that context can be cleared
